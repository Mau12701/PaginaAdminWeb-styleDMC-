Glosario de GitLab y Git

ConceptoDescripción
GitLabPlataforma de desarrollo de software con gestión de código fuente y CI/CD.
DevOpsMetodología que une desarrollo y operaciones para mejorar la entrega de software.
CI/CDAutomatización de la integración y despliegue continuo de software.
GitSistema de control de versiones distribuido.
GitLab FlowFlujo de trabajo basado en Git para colaboración eficiente.
Git RepositoryRepositorio donde se almacena el código y su historial de cambios.
Git InitComando para inicializar un nuevo repositorio de Git.
Git StatusMuestra el estado actual del repositorio.
Git AddMueve archivos al área de preparación antes de confirmar cambios.
Git CommitGuarda los cambios en el historial del repositorio.
Git LogMuestra un historial de los commits realizados.
Git BranchCrea una nueva rama en el repositorio.
Git CheckoutCambia la rama activa en el repositorio.
Git MergeFusiona una rama en otra.
Git StashGuarda temporalmente cambios sin comprometerlos.
Grupos en GitLabPermiten gestionar configuración y permisos de múltiples proyectos.
Proyectos en GitLabContenedores que almacenan repositorios con CI/CD.
Miembros en GitLabUsuarios o grupos con acceso a un proyecto.
Merge RequestsSolicitudes de fusión para revisar cambios antes de integrarlos.
Issues en GitLabHerramienta para rastrear tareas y errores.
JenkinsHerramienta de automatización para CI/CD.
LambdaTestPlataforma para pruebas automatizadas en múltiples navegadores.
Empaquetado y liberación en GitLabFuncionalidades para distribuir software eficientemente.
